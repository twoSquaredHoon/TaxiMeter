# TaxiMeter

A touch-first **taxi meter phone UI prototype** built with React and Vite. It simulates a Korean taxi meter: status banner, live fare/countdown display, and a 2×2 control pad for the four meter modes.

## Quick start

```bash
npm install
npm run dev
```

Open the URL from the terminal (usually [http://localhost:5173](http://localhost:5173)).

| Command | What it does |
| --- | --- |
| `npm run dev` | Starts the Vite dev server with hot reload. Use this while developing. |
| `npm run dev:test` | Opens the **test** page with manual speed controls. |
| `npm run build` | Bundles the app into static files under `dist/` for deployment. |
| `npm run preview` | Serves the `dist/` build locally so you can test production output. |

### Test mode (manual speed)

Open [http://localhost:5173/test/](http://localhost:5173/test/) (or run `npm run dev:test`).

A **bottom-right test menu** (test-only files under `src/test/`) lets you override:

- **속도** — presets `0–80` km/h, custom input, **자동 속도**
- **퍼센트** — presets `0–100%`, custom input, **자동 %**

Horse gallop speed follows the % on screen (slow → normal → fast). The main app at `/` is unchanged.

## How the app behaves

The UI is a dark portrait “phone” frame with three regions:

1. **Status banner** — shows the current mode label (`빈차`, `주행중`, `할증`, `지불`).
2. **Meter panel** — horse icon + progress %, trip number, speed, fare (원), magenta countdown, secondary value.
3. **Control grid** — four large buttons in a 2×2 layout.

### Control buttons

| Button | Screen | Behavior |
| --- | --- | --- |
| **빈차** (empty) | `EmptyScreen` | Stops simulation. Resets fare to `3000` and countdown to `2000`. Dims the meter. Speed shows `0.0 Km/h`. Horse shows `0%`. |
| **주행 (시외)** (driving) | `DrivingScreen` | Starts simulation. Countdown ticks down; at `0`, fare increases and countdown resets. Speed wobbles around ~42 km/h. |
| **할증** (surcharge) | `SurchargeScreen` | Toggles surcharge on/off while driving. Fare increments by 1.5× when on. Banner shows `할증`; fare uses pink styling. If pressed from empty, switches to driving first. |
| **지불** (payment) | `PaymentScreen` | Stops simulation. Keeps current fare. Banner turns green-tinted. Speed shows `0.0 Km/h`. |

### Simulation (driving mode only)

On each countdown tick (every `1` second):

- Countdown decreases by `100`.
- When countdown hits `0`, fare increases by `100` (or `150` with surcharge) and countdown resets to `2000`.
- Horse progress % is derived from how far the countdown has dropped toward zero.

Tunable values live in `src/config.js`.

### Root files

| File | Purpose |
| --- | --- |
| `index.html` | The only HTML file. Loads Google fonts (Noto Sans KR, Share Tech Mono), defines `#root`, and points Vite at `src/main.jsx`. |
| `vite.config.js` | Minimal Vite setup with `@vitejs/plugin-react` so `.jsx` files compile and hot-reload in dev. |
| `package.json` | Declares React 19, Vite 6, and the three npm scripts above. |
| `.gitignore` | Keeps `node_modules/` and `dist/` out of git — both are regenerated locally. |

### `src/` — application code

| File | Purpose |
| --- | --- |
| `main.jsx` | Creates the React root and renders `<App />` inside `<StrictMode>`. |
| `App.jsx` | Owns the outer `device-frame`, calls `useTaxiMeter()`, and renders `<MeterScreen />` plus the hint line below the phone. |
| `index.css` | CSS variables (colors, spacing), phone frame layout, meter glow styles, per-state modifiers (`.state-empty`, `.state-surcharge`, `.state-payment`), and responsive tweaks. |
| `config.js` | `CONFIG` — numeric simulation settings. `STATUS_LABELS` — Korean strings for each mode. |

### `src/hooks/useTaxiMeter.js`

Central state for the prototype. Exposes everything screens and buttons need:

- **State:** `mode`, `surcharge`, `fare`, `countdown`, `speed`
- **Derived:** `statusText`, `horsePercent`, `appClassName` (CSS state classes), `activeButton`
- **Actions:** `handleEmpty`, `handleDriving`, `handleSurcharge`, `handlePayment`
- **Timers:** `setInterval` tick loop only runs when `mode === "driving"`

### `src/screen/` — one view per button

`screen/index.jsx` picks which screen to show:

1. `mode === "empty"` → `EmptyScreen`
2. `mode === "payment"` → `PaymentScreen`
3. `surcharge === true` → `SurchargeScreen` (while still in driving mode)
4. otherwise → `DrivingScreen`

Each screen file describes **what that button’s UI looks like**. They all use the same shared building blocks from `screen/shared/` but pass different values (e.g. empty screen forces `0%` and `0.0 Km/h`).

### `src/screen/shared/` — reusable UI

| Component | Renders |
| --- | --- |
| `ScreenLayout` | `<main>` with status banner, screen-specific meter content, and control buttons. |
| `StatusBanner` | The top status strip and label text. |
| `MeterSection` | The flex container for the right-aligned meter stack. |
| `MeterHorseBlock` | Horse icon + percent label. |
| `HorseIcon` | Inline SVG of the horse. |
| `MeterMeta` | Trip number (default `34`) and speed. |
| `MeterFare` | Fare digits + `원`. |
| `MeterBottom` | Magenta countdown and green secondary value. |
| `ControlButtons` | The four mode buttons; wired to handlers from `useTaxiMeter`. |

## Configuration reference

Edit `src/config.js` to change simulation behavior:

| Key | Default | Meaning |
| --- | --- | --- |
| `initialFare` | `3000` | Fare after reset (빈차 / 주행). |
| `fareIncrement` | `100` | Added each time countdown hits zero. |
| `countdownStart` | `2000` | Countdown reset value. |
| `countdownStepMs` | `1000` | Milliseconds between countdown ticks (1 second). |
| `countdownDecreasePerTick` | `100` | Countdown drop per tick (`2000` → `1900` → …). |
| `speedUpdateMs` | `120` | How often simulated speed (and horse pace) refresh. |
| `simulatedSpeedKmh` | `42` | Base speed shown in driving mode (with small wobble). |
